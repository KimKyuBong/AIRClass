### 3. Backup & Restore System âœ…
**Files**: 
- `scripts/backup-cluster.sh` (145 lines)
- `scripts/restore-cluster.sh` (140 lines)

**What it does**:
- **Backup**: Configuration, cluster state, Docker status
- **Restore**: One-command disaster recovery
- **Automated**: Cron-friendly with retention policy
- **Compressed**: tar.gz archives

**Backs up**:
- Environment configuration (.env)
- Docker compose files
- MediaMTX configuration
- Cluster state snapshot (from /cluster/nodes)
- Container status
- Volume information

**Usage**:

**Manual Backup**:
```bash
./scripts/backup-cluster.sh
# Output: backups/airclass_backup_20240122_180000.tar.gz
```

**Automated Backup** (keeps last 7 days):
```bash
./scripts/backup-cluster.sh --auto

# Add to crontab for daily backups at 2 AM
crontab -e
0 2 * * * /opt/airclass/scripts/backup-cluster.sh --auto
```

**Restore**:
```bash
# From directory
./scripts/restore-cluster.sh backups/airclass_backup_20240122_180000

# From archive
./scripts/restore-cluster.sh backups/airclass_backup_20240122_180000.tar.gz
```

**Safety features**:
- Confirmation prompt before restore
- Stops running services automatically
- Displays backup metadata
- Cleans up temp files

### 4. Prometheus Metrics Integration âœ…
**Files Modified**: 
- `backend/main.py` (added metrics endpoint + tracking)
- `backend/requirements.txt` (added prometheus-client)

**Endpoint**: `GET /metrics`

**Metrics Exposed**:

**HTTP Metrics**:
- `airclass_http_requests_total{method, endpoint, status}` - Request counts
- `airclass_http_request_duration_seconds{method, endpoint}` - Latency histogram

**Streaming Metrics**:
- `airclass_active_streams` - Number of active RTMP streams
- `airclass_active_connections{type}` - WebSocket/HLS connections by type

**Token Metrics**:
- `airclass_tokens_issued_total{user_type}` - JWT tokens issued (teacher/student/monitor)

**Cluster Metrics**:
- `airclass_cluster_nodes_total{status}` - Nodes by status (active/offline)
- `airclass_cluster_load_percentage{node_id}` - Per-node load %
- `airclass_cluster_connections{node_id}` - Per-node connection count

**Error Metrics**:
- `airclass_errors_total{type}` - Errors by type (auth/stream/cluster/websocket)

**Usage**:
```bash
# View metrics
curl http://localhost:8000/metrics

# Example output:
# HELP airclass_active_connections Number of active connections
# TYPE airclass_active_connections gauge
airclass_active_connections{type="teacher"} 1.0
airclass_active_connections{type="student"} 45.0
airclass_active_connections{type="monitor"} 3.0

# HELP airclass_tokens_issued_total Total JWT tokens issued
# TYPE airclass_tokens_issued_total counter
airclass_tokens_issued_total{user_type="student"} 123.0
airclass_tokens_issued_total{user_type="teacher"} 5.0
```

**Integration with Prometheus**:
```yaml
# prometheus.yml
scrape_configs:
  - job_name: 'airclass'
    static_configs:
      - targets: ['master:8000', 'slave-1:8000', 'slave-2:8000']
    metrics_path: '/metrics'
    scrape_interval: 15s
```

**Grafana Queries**:
```promql
# Total cluster capacity
sum(airclass_cluster_connections)

# Average load across nodes
avg(airclass_cluster_load_percentage)

# Request rate
rate(airclass_http_requests_total[5m])

# Active users
sum(airclass_active_connections)
```
